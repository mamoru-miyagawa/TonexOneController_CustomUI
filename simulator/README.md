# Tonex Controller — desktop simulator

Renders the 480×320 main screen on a desktop window so you can iterate on the
UI without flashing the Waveshare 3.5B. Same `screens.c` runs here and on the
device; differences are confined to `lv_conf.h` (PC vs ESP-IDF Kconfig) and a
thin SDL bridge in `main.c`.

## What's where

| File | Role |
|---|---|
| `main.c` | SDL2 window + LVGL display/input bridge + tick loop |
| `lv_conf.h` | LVGL config (color depth, fonts, theme) mirroring `sdkconfig.ws35b` |
| `stub_actions.c` | Empty `action_*` handlers; PREV/NEXT cycle 8 demo presets |
| `CMakeLists.txt` | FetchContent pulls LVGL v8.3.11, links SDL2 via pkg-config |

The simulator builds `../source/main/ui_handwritten_480x320land/` (the
firmware's edit target). The EEZ-managed
`../source/main/ui_generated_480x320land/` is **not** in this build.

## Prerequisites (MSYS2 / MinGW64)

Install MSYS2 once: <https://www.msys2.org/>. Open the **MinGW64 shell** (not
the plain MSYS shell), then:

```bash
pacman -Syu                                    # may need a second run
pacman -S --needed \
    mingw-w64-x86_64-toolchain \
    mingw-w64-x86_64-cmake \
    mingw-w64-x86_64-ninja \
    mingw-w64-x86_64-SDL2 \
    mingw-w64-x86_64-pkgconf \
    git
```

## Build

```bash
cd simulator
cmake -S . -B build -G Ninja
cmake --build build
```

First build pulls LVGL v8.3.11 from GitHub (~30 s on a decent connection) and
compiles ~120 image .c files plus screens.c. Roughly 30–60 s total.

Incremental rebuilds after editing `../source/main/ui_handwritten_480x320land/screens.c`
take ~1 second.

## Run

```bash
./build/tonex_sim.exe
```

If the window opens but the title bar shows a freeze, MinGW may have linked
against the wrong SDL2.dll — make sure `/mingw64/bin/SDL2.dll` is on `PATH` or
copied next to the .exe.

### Controls

| Input | Effect |
|---|---|
| Click any chip | logs which effect was clicked |
| PREV / NEXT buttons | cycles through 8 demo presets (preset name, bank pill, BPM, chip on/off update) |
| Long-press preset name | logs "skin edit" (no-op in sim) |
| Click settings gear | logs "settings clicked" |
| Swipe | logs gesture direction |
| ESC | quit |

## Iterating on the design

1. Edit `../source/main/ui_handwritten_480x320land/screens.c` (or `tonex_theme.h`)
2. `cmake --build build` (≈1 s)
3. `./build/tonex_sim.exe`

The handwritten folder is **not regenerated by EEZ Studio**. Anything you
change there sticks. If you want to keep using EEZ for design exploration,
work in `ui_design_480x320land/Tonex_Controller_480_320.eez-project` and
manually port any changes you like into the handwritten folder.

## Known gaps

- **Touch is single-point mouse only.** Multi-touch gestures (pinch, two-finger
  swipe) aren't simulated. The device's gesture detection still works through
  `action_gesture` for single-finger swipes.
- **No haptic / no real preset data.** PREV/NEXT cycle a hardcoded table in
  `stub_actions.c` — they don't call the real `control_request_preset_*`
  paths because those depend on USB / FreeRTOS / the actual pedal.
- **Effect detail screens use the legacy tabview UI** (untouched by the
  foundation port). Clicking the gear logs but doesn't navigate yet —
  `action_show_settings_page` is a no-op in the sim.
- **Color may render slightly differently** than on the device. The device
  uses BGR with `LV_COLOR_16_SWAP=1`; the sim uses RGB565 native via SDL.
  Hues should match; if they don't, that's a config mismatch worth flagging.

## When something goes wrong

| Symptom | Probable cause | Fix |
|---|---|---|
| `pkg-config not found` | missing pkgconf | `pacman -S mingw-w64-x86_64-pkgconf` |
| `SDL2 not found` | wrong shell | open the **MinGW64** shell, not MSYS or UCRT |
| Black window, no UI | LVGL display driver not registered | check `lv_disp_drv_register` ran before `ui_init()` |
| Linker can't find `lv_font_montserrat_N` | screens.c uses a font the sim's `lv_conf.h` didn't enable | flip the matching `LV_FONT_MONTSERRAT_N` in `lv_conf.h` to `1` |
| FetchContent fails | no network or git missing | `pacman -S git`; rerun `cmake -S . -B build` |
